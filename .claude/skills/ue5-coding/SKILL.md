## 코드 작업 워크플로우

모든 코드 수정/작성 요청에서 반드시 아래 순서를 따른다.

### 1. 작업 전 파일 먼저 읽기
- 프로젝트 루트의 `.clang-format` 파일을 Read 툴로 읽고 해당 규칙을 파악한다
- 수정할 파일을 Read 툴로 반드시 먼저 읽는다
- 연관된 헤더(.h)와 구현(.cpp) 파일을 함께 파악한다
- 기존 패턴과 구조를 이해한 후 작업을 시작한다

### 2. 변경 범위 최소화
- 요청된 것만 수정한다. 요청하지 않은 리팩토링, 정리, 개선은 하지 않는다
- 불필요한 주석, docstring, 타입 어노테이션을 추가하지 않는다
- 관련 없는 코드는 건드리지 않는다

### 3. 코딩 컨벤션 준수
- `.clang-format`에서 읽은 규칙을 그대로 따른다
- UE5 네이밍 컨벤션 준수: 클래스 접두사(A/U/F/E/I), PascalCase 함수/변수
- UE5 매크로(UPROPERTY, UFUNCTION, UCLASS, USTRUCT, UENUM, UINTERFACE, GENERATED_BODY)는 별도 줄에 단독 배치

### 4. 헤더 주석
- 새 파일을 생성할 때: 파일 상단에 해당 파일의 역할을 한 줄 주석으로 추가한다
- 새 함수/클래스를 헤더에 추가할 때: 선언 위에 무엇을 하는지 한 줄 짧은 설명 주석을 추가한다
- 기존 코드에는 주석을 추가하지 않는다 (요청하지 않은 경우)
