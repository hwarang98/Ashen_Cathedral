# Issue #3 PR 작업 단위 하위 이슈 제안

> 참고: 현재 로컬 저장소에서는 GitHub 원본 `Issue #3` 본문을 직접 조회할 수 없어,
> 이 프로젝트의 구조(UE5 + GAS + Combat)를 기준으로 **PR 단위로 병합 가능한 하위 이슈**로 쪼갰습니다.

## 목표
- 큰 이슈를 작은 단위로 분해해 리뷰/테스트/롤백 부담을 줄입니다.
- 각 하위 이슈는 **독립적으로 머지 가능한 PR 1개**를 목표로 합니다.

## 하위 이슈 목록 (PR 단위)

### 3-1. GameplayTag 정리 및 네이밍 표준화
- **범위**
  - 신규/기존 태그 네임스페이스 정리 (`Input`, `Player`, `Shared`)
  - 하드코딩 문자열 제거 및 상수 접근으로 통일
- **완료 조건**
  - [ ] 태그 선언/정의 파일 간 일관성 유지
  - [ ] 컴파일 오류 없음
- **PR 제목 예시**
  - `refactor(tags): gameplay tag 네이밍/구조 정리`

### 3-2. AttributeSet 기본 스탯(체력/스태미나) 정합성 보강
- **범위**
  - `ACAttributeSet`의 기본 스탯 clamp/초기값 처리 강화
  - 값 변경 시점 로깅 또는 검증 코드 보강
- **완료 조건**
  - [ ] 음수/최대치 초과 상태 방지
  - [ ] 에디터/PIE에서 스탯 반영 확인
- **PR 제목 예시**
  - `feat(gas): AttributeSet 스탯 검증/보정 로직 추가`

### 3-3. 스프린트 Ability 발동/종료 플로우 안정화
- **범위**
  - `ACAbility_Sprint` 활성/종료 조건 정리
  - 스태미나 부족 시 종료 처리 및 중복 발동 방지
- **완료 조건**
  - [ ] 스프린트 시작/종료 이벤트 정상 동작
  - [ ] 엣지 케이스(연타 입력/자원 부족)에서 오동작 없음
- **PR 제목 예시**
  - `fix(ability): sprint ability 상태 전이 안정화`

### 3-4. 입력 바인딩 & InputConfig 매핑 검증
- **범위**
  - `ACInputComponent`, `ACDataAsset_InputConfig`의 액션 매핑 검증
  - 스프린트/장착/해제 입력 충돌 점검
- **완료 조건**
  - [ ] 모든 주요 액션 바인딩 확인
  - [ ] 잘못된 매핑 시 안전한 실패 처리
- **PR 제목 예시**
  - `chore(input): InputConfig 매핑 검증 및 안정화`

### 3-5. 전투 컴포넌트와 Ability 연계 테스트 시나리오 추가
- **범위**
  - `PlayerCombatComponent`와 무기 장착/해제 Ability 연계 검증 포인트 문서화
  - 최소 재현 시나리오(PIE 절차) 정리
- **완료 조건**
  - [ ] 수동 테스트 절차 문서 1건 이상
  - [ ] 회귀 확인 체크리스트 포함
- **PR 제목 예시**
  - `docs(test): combat-ability 연계 테스트 시나리오 추가`

### 3-6. 개발 문서(README/가이드) 업데이트
- **범위**
  - 변경된 태그/어빌리티/입력 플로우 문서 반영
  - 신규 기여자를 위한 실행/검증 절차 업데이트
- **완료 조건**
  - [ ] README 또는 Docs 문서에 반영
  - [ ] 변경사항 요약 섹션 포함
- **PR 제목 예시**
